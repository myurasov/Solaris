## Summary

0.4.0 is a terminology rename - no behavior change. The project persona is renamed **`developer` ->
`engineer`** (so `ai/developer.agent.md` -> `ai/engineer.agent.md` and `ai/developer.instructions.md` ->
`ai/engineer.instructions.md`), and the materialized project layer is renamed **"ai-setup" -> "ai-pack"**
throughout the prose. Framework-side this release also renames `solaris/templates/ai-setup/` -> `ai-pack/`
and the `version` tool's `aisetup` subcommand -> `aipack`, but those need no per-project action. Two
conventions become explicit (a well-formed project already satisfies both): `ai/spec.md` is **self-sufficient**
(reads standalone, references no other file), and **every change to a revisioned file increments its rev**.

## Pre-flight checks

- [ ] `projects/<slug>/ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.4.0 (`uv run -m solaris.tools.version current`).
- [ ] `projects/<slug>/ai/developer.agent.md` exists (else the rename was already done - skip step 1).

## Migrate

1. **Rename the persona files** (only if not already renamed; `git mv` when an ai/ or src/ git tracks them,
   otherwise a plain move):
   - `projects/<slug>/ai/developer.agent.md` -> `projects/<slug>/ai/engineer.agent.md`
   - `projects/<slug>/ai/developer.instructions.md` -> `projects/<slug>/ai/engineer.instructions.md`
2. **Term rename inside the ai-pack.** In the project's `AGENTS.md` and every `ai/*.md` + `ai/**/*.md`
   **except** `ai/<plugin>/` overlays, apply, in order: `ai-setup` -> `ai-pack`, `aisetup` -> `aipack`,
   `Developer` -> `Engineer`, `developer` -> `engineer`. This also fixes the renamed filenames referenced in
   prose (e.g. `developer.agent.md` -> `engineer.agent.md`). Leave `ai/<plugin>/` overlays to their plugin's
   own migration.
3. **Reconcile revisions.** The materialized master path changed
   (`ai/developer.agent.md` -> `ai/engineer.agent.md`); `update-project`'s revisions step
   (`revs classify` / `ff` then `baseline`) reconciles it. Bump the rev of any rev-marked `ai/` file you
   edited (`uv run -m solaris.tools.revs bump projects/<slug>/ai/engineer.agent.md`).
4. **Stamp the version:** `uv run -m solaris.tools.version set --dir projects/<slug> 0.4.0`.

## Revert

1. `uv run -m solaris.tools.version set --dir projects/<slug> 0.3.0`.
2. Rename back: `ai/engineer.agent.md` -> `ai/developer.agent.md`,
   `ai/engineer.instructions.md` -> `ai/developer.instructions.md`.
3. Reverse the term replacements in `AGENTS.md` + `ai/` (non-plugin): `ai-pack` -> `ai-setup`,
   `aipack` -> `aisetup`, `Engineer` -> `Developer`, `engineer` -> `developer`.

## Summary

0.11.0 moves blocked-command wrappers out of the in-repo, gitignored `.tools/` and into `/tmp`. When a CLI
tool is blocked (sandbox / permission policy / subscription / otherwise), the agent still creates a
reversed-name `#!/bin/sh` `exec <tool> "$@"` pass-through (`open` -> `nepo`, `ssh` -> `hss`) and registers
it, but now writes it to `/tmp/<name>` and invokes it as `/tmp/<name>`. This keeps the workaround fully
outside the working tree (no gitignore entry needed) and shared across project roots.

Additive and content-only: the ai-pack `engineer.agent.md` template's `## Blocked-command wrappers` section
is reworded for the new location. No ai-pack schema or file layout changed; the update reaches a project
through the normal per-file revisions fast-forward.

## Pre-flight checks

- [ ] `ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.11.0 (`uv run -m solaris.tools.version current`).

## Migrate

1. **Fast-forward the revisioned files:** `uv run -m solaris.tools.revs ff --dir <project>`. This brings
   `ai/engineer.agent.md` up to the current master rev (the `## Blocked-command wrappers` section now points
   at `/tmp`). Files the user never edited move forward automatically; resolve any `merge-up`/`conflict` per
   `update-project` first.
2. **Stamp the version:** `uv run -m solaris.tools.version set --dir <project> 0.11.0`.
3. **Re-baseline:** `uv run -m solaris.tools.revs baseline --dir <project>`.
4. **(Optional) move any existing wrappers:** if the project's `ai/memory/` registered wrappers under a
   project-local `.tools/`, recreate them in `/tmp` and update the registry; delete the old `.tools/`.

## Validate

- [ ] `ai/manifest.json` `framework_version` is `0.11.0`.
- [ ] `ai/engineer.agent.md`'s `## Blocked-command wrappers` section references `/tmp`, not `.tools/`.

## Revert

1. `uv run -m solaris.tools.version set --dir <project> 0.10.0`.
2. The wording change is harmless if left in place; to fully revert, restore the prior rev of
   `ai/engineer.agent.md` from git history and `uv run -m solaris.tools.revs baseline --dir <project>`.

## Summary

0.6.0 renames the **local-mode code directory** from `src/` to `source/` (so `projects/<slug>/source/`). Pure
rename, no behavior change: the engineer's working directory, the path `--remote` rsyncs, and where the
project's own `git init` runs all become `source/`. Remote-code projects (which have no `src/`, only a
`remote.json`) are unaffected. Application code that derives paths from `__file__` keeps working unchanged;
only references that hard-code the literal `src/` need updating.

0.6.0 also adds a new **opt-in `embedded` project mode** (the ai-pack lives inside the source repo at
`projects/<slug>/<repo>/`, no separate `source/`). It is additive - existing projects need no action; the mode
is chosen only for new or freshly-imported projects.

## Pre-flight checks

- [ ] `projects/<slug>/ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.6.0 (`uv run -m solaris.tools.version current`).
- [ ] The project is `local` mode with a `projects/<slug>/src/` dir (else nothing to rename - skip to step 4).

## Migrate

1. **Rename the directory:** `projects/<slug>/src` -> `projects/<slug>/source` (`git mv` if the project's own
   git tracks it, else a plain move). The directory keeps its own `.git` (git repos relocate cleanly);
   gitignored envs (`.venv`) hold absolute paths and break on move - recreate them on the next `uv sync`.
2. **Update references to the literal `src/`** in `projects/<slug>/AGENTS.md` and under `ai/` (engineer agent,
   instructions, memory): the project-code-dir `src/` -> `source/`, and `cd src` -> `cd source`. **Do not**
   touch a nested framework's own dir (e.g. Angular's `ui/src/`), HTML `src="..."` attributes, or code
   identifiers (e.g. a `SRC_DIR` constant resolved from `__file__` - it keeps working under the new name).
3. **Fix any hard-coded `src/` in the app code** only where it does not derive from `__file__` (most code does
   not need this). Then rebuild the local env: `cd source && uv sync` (the old `.venv` path is stale).
4. **Stamp the version:** `uv run -m solaris.tools.version set --dir projects/<slug> 0.6.0`.

## Validate

- [ ] `projects/<slug>/source/` exists; `projects/<slug>/src/` does not.
- [ ] No project-code-dir `src/` reference remains in `AGENTS.md` or under `ai/` (a nested `ui/src/` is fine).
- [ ] `ai/manifest.json` `framework_version` is `0.6.0`.
- [ ] The app still runs from `source/` (`cd source && uv run ...`).

## Revert

1. `uv run -m solaris.tools.version set --dir projects/<slug> 0.5.0`.
2. Rename back `projects/<slug>/source` -> `projects/<slug>/src`, and reverse the `source/` -> `src/` and
   `cd source` -> `cd src` reference edits in `AGENTS.md` + `ai/` (leaving any nested `ui/src/` untouched).

## Summary

0.7.0 renames the private/local working-context file **`ai/memory/info.md` -> `ai/memory/context.md`** and
redefines it as a **verbose, model-facing context log**: richer than `interactions.jsonl`, it captures the
model's own answers/decisions/findings in prose. It has a curated **Standing context** section (what the
project is, code map, run/deploy, gotchas) that survives compaction, a newest-first **Log**, and a
**Previous History** of compacted older entries once the Log grows past **~100KB**. Only the engineer and
Solaris's own agents write it. Pure rename + restructure of a gitignored private file - no behavior change
to the app; a project that never had an `info.md` just gets a fresh `context.md` from the template.

## Pre-flight checks

- [ ] `<project>/ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.7.0 (`uv run -m solaris.tools.version current`).
- [ ] Note: `ai/memory/` is gitignored (and, in embedded mode, excluded by the repo's own `.gitignore`), so
      the rename in step 1 is a plain `mv`, not `git mv`.

## Migrate

1. **Rename (or create) the file.** If `<project>/ai/memory/info.md` exists, `mv` it to
   `<project>/ai/memory/context.md`. If it does not, copy the template
   `solaris/templates/ai-pack/ai/memory/context.md` to `<project>/ai/memory/context.md` and substitute
   `{{NAME}}` from `ai/manifest.json` - then stop (no content to migrate).
2. **Restructure into the context-log shape.** Give the file an H1 (`# <NAME> - context <!-- omit in toc -->`)
   and the `## How to use this file` preamble from the template, move the existing curated content under a
   `## Standing context` heading, and add empty `## Log` and `## Previous History` sections. Then refresh the
   TOC: `uv run -m solaris.tools.toc --write <project>/ai/memory/context.md`.
3. **Update references in `ai/`.** In `ai/engineer.agent.md`: add `ai/memory/context.md` to "Context to load"
   and add the context-log rule to "Memory" (append a verbose entry per turn, newest-first, ~100KB ->
   Previous History, engineer + Solaris agents only). In `ai/engineer.instructions.md`: change any `info.md`
   mention to `context.md`. In `ai/manifest.json`: update any `info.md` reference (e.g. an
   `info.imported_from` pointer).
4. **Re-record the revisions baseline** (the engineer agent changed):
   `uv run -m solaris.tools.revs baseline --dir <project>`.
5. **Stamp the version:** `uv run -m solaris.tools.version set --dir <project> 0.7.0`.

## Validate

- [ ] `<project>/ai/memory/context.md` exists with `## Standing context`, `## Log`, and `## Previous History`;
      `ai/memory/info.md` is gone.
- [ ] No `info.md` reference remains under `ai/` (engineer agent, instructions, manifest).
- [ ] `uv run -m solaris.tools.toc --check <project>/ai/memory/context.md` is clean.
- [ ] `ai/manifest.json` `framework_version` is `0.7.0`.

## Revert

1. `uv run -m solaris.tools.version set --dir <project> 0.6.1`.
2. `mv <project>/ai/memory/context.md <project>/ai/memory/info.md` and collapse the `## Standing context`
   content back to the top (drop the `## How to use this file` / `## Log` / `## Previous History` scaffolding);
   refresh its TOC.
3. Reverse the reference edits in `ai/engineer.agent.md`, `ai/engineer.instructions.md`, and
   `ai/manifest.json` (`context.md` -> `info.md`), then `revs baseline --dir <project>`.

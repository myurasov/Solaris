## Summary

0.13.0 is a **framework-additive** release; no ai-pack schema, layout, or revisioned ai-pack file changed,
so this is a marker migration (it only records the version step). Two things changed at the framework root:

1. **New bundled plugin `plugins/visual-qa/`** - a GPU-agnostic "eyes" for visual end-to-end testing: a
   pluggable vision-language model behind an OpenAI-compatible endpoint, `look` / `assert_visual` tools
   (an MCP server + CLI in `shared/eyes.py`), a GPU-aware model recommender (`detect` / `recommend` over
   `shared/models.json`, ranked by VRAM + architecture + task), and a `serving/` runbook (vLLM / NIM /
   Ollama, verified on a DGX Spark GB10). Attach it to a project with `install-plugin` like any plugin.
2. **Plugin tracking model.** The blanket `plugins/*` gitignore is removed; a `plugins/.gitignore` now
   ignores only plugins that are their **own git repository** (e.g. `nvidia-isaac-lab/`). Plugins authored
   in-place (no own repo) are tracked in the framework repo. Each plugin still keeps its own
   `revisions.json`; the framework ledger never tracks plugin files.

Existing projects need no file changes. Attaching `visual-qa` is opt-in per project via `install-plugin`.

## Pre-flight checks

- [ ] `ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.13.0 (`uv run -m solaris.tools.version current`).

## Migrate

1. **Stamp the version:** `uv run -m solaris.tools.version set --dir <project> 0.13.0`.
2. **Re-baseline** (no-op for content, keeps the ledger tidy): `uv run -m solaris.tools.revs baseline --dir <project>`.
3. **(Optional) attach visual-qa** to a project that needs visual E2E: "add plugin visual-qa to <project>"
   (`install-plugin`), then stand up an endpoint per `plugins/visual-qa/serving/README.md`.

## Validate

- [ ] `ai/manifest.json` `framework_version` is `0.13.0`.
- [ ] `uv run -m solaris.tools.version check --dir <project>` passes.

## Revert

1. `uv run -m solaris.tools.version set --dir <project> 0.12.1`.
2. Nothing else to undo: no revisioned ai-pack file changed. If `visual-qa` was attached, detach it by
   removing `ai/visual-qa/`, its `plugins[]` entry in `ai/manifest.json`, and its `mcpServers` entries.

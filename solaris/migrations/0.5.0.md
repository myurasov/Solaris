## Summary

0.5.0 makes three things explicit, all additive / non-breaking:

1. **`ai/manifest.json` carries only project metadata + versions** - any host/deploy/port/URL/secret specifics
   move out of the manifest `info` block into `ai/memory/resources.md` (or `credentials.md`).
2. **The engineer bootstraps `ai/memory/` interactively** when it is missing or empty (e.g. an ai-pack shared
   without its private layer). This arrives via the updated `ai/engineer.agent.md` (the revisions step below),
   so it needs no manual edit.
3. **Plugins keep their own revision ledger** at `plugins/<name>/revisions.json`; the framework
   `solaris/revisions.json` tracks only framework masters. This is framework/plugin-side - no project action.

## Pre-flight checks

- [ ] `projects/<slug>/ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.5.0 (`uv run -m solaris.tools.version current`).

## Migrate

1. **Revisions sync first.** `update-project`'s revisions step brings `ai/engineer.agent.md` to the new master
   rev (which carries the "bootstrap `ai/memory/` when absent" instruction). No manual edit.
2. **Move host/deploy specifics out of the manifest.** If `ai/manifest.json` has an `info` block with any
   host/deploy/port/URL value (e.g. `deploy_target`, `host`, `port`, `url`), relocate each into
   `ai/memory/resources.md` (the "Deploy / run target" section) and any secret into `ai/memory/credentials.md`,
   then delete those keys from the manifest. Keep only non-host metadata in `info` (e.g. `product_name`), or
   drop `info` entirely if it becomes empty. Nothing is lost - every value lands in `ai/memory/`.
3. **Stamp the version:** `uv run -m solaris.tools.version set --dir projects/<slug> 0.5.0`.

(Plugins, if any: run `uv run -m solaris.tools.revs ledger` once - it writes each
`plugins/<name>/revisions.json` and drops plugin entries from the framework ledger. This is repo-wide, not
per-project.)

## Validate

- [ ] `ai/manifest.json` `info` has no host/deploy/port/URL/secret keys; each relocated value is present in
      `ai/memory/resources.md` (or `credentials.md`).
- [ ] `ai/manifest.json` `framework_version` is `0.5.0`.
- [ ] `uv run -m solaris.tools.revs classify --dir projects/<slug>` reports `ai/engineer.agent.md` in-sync (or
      `conflict` only if you intentionally customized it).

## Revert

1. `uv run -m solaris.tools.version set --dir projects/<slug> 0.4.1`.
2. Optionally re-add the host/deploy fields to `ai/manifest.json` `info` (the values remain in
   `ai/memory/resources.md` either way). The bootstrap instruction reverts when `ai/engineer.agent.md` is
   synced back to the 0.4.1 master rev.

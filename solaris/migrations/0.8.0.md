## Summary

0.8.0 adds a raw **`prompt`** field to interaction-log entries. Each agent-authored line is now
`{ts, project, prompt, request, outcome}` - `prompt` is the user's verbatim raw prompt, `request` is the
agent's interpreted restatement of it (the field is unchanged in name and meaning), and `outcome` is the
result. The agent writes this full entry identically into the framework master `memory/interactions.jsonl`
and the touched project's `ai/memory/interactions.jsonl`; the prompt-submit hook still appends a
`{ts, cwd, ide, prompt}` raw-prompt backstop line to the master. Purely **additive**: existing log lines (no
`prompt`) stay valid, so there is no data rewrite - only the schema text in `ai/engineer.agent.md` changes,
and new entries carry the field.

## Migrate

1. **Sync the engineer agent.** `ai/engineer.agent.md` is revision-tracked, so reconcile it with the master:
   `uv run -m solaris.tools.revs classify --dir <project>` then `revs ff --dir <project>` (fast-forward) for
   an untouched copy, or a smart merge if the user customized it. The Memory section now documents the
   `{ts, project, prompt, request, outcome}` schema and the `prompt` field.
2. **No log-data migration.** Leave existing `ai/memory/interactions.jsonl` lines as-is (additive change).
   From now on, write each entry as `{ts, project, prompt, request, outcome}`.
3. **Stamp the version:** `uv run -m solaris.tools.version set --dir <project> 0.8.0`.

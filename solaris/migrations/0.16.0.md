## Summary

Adds a second, opt-in plugin install mode: **link**. Instead of copying a plugin's `shared/` into
`ai/<name>/`, `install-plugin` writes a self-describing pointer file `ai/<name>.link.md` that names the
live `plugins/<name>/` source; the manifest entry becomes `{name, "mode": "link"}` (no `version` - a
linked plugin always runs the live source). The revs tools skip linked entries and `version check-plugins`
reports them as live. Purely additive: existing copy installs are untouched, and no project gains a linked
plugin by migrating. The ai-pack templates (`AGENTS.md` rev 8, `engineer.agent.md` rev 17) now instruct
the engineer to follow link files; they fast-forward via the routine revisions sync.

   0.15.1 masters) and set `framework_version` back to `0.15.1`. Any `ai/<name>.link.md` created after
   migrating can be converted to a copy install via `install-plugin` ("install plugin <name> to
   <project>") before reverting.

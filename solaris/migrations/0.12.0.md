## Summary

0.12.0 is a **framework-internal** release; no ai-pack schema, layout, or revisioned file changed, so this is
a marker migration (it only records the version step). Two things changed at the framework root: (1) a new
stdlib tool `solaris.tools.read_first` now auto-injects the AGENTS.md read-first set (orchestrator role,
commit + safety rules, `memory/instructions.md`) at session start via a hook - wired to Claude Code's
`SessionStart` (full load) + `UserPromptSubmit` (a `--remind` one-liner) and Cursor's `sessionStart`, with
IDE-aware output (Cursor JSON `additional_context` vs Claude plain stdout). (2) The ai-pack templates
`ai/memory/resources.md` and `ai/engineer.instructions.md` are reworded so `resources.md` is **inventory
only** (hardware + hosts/accounts - the *what exists*) and all procedures / runtime details / gotchas (*how*)
live in `engineer.instructions.md`.

Existing projects need no file changes: the two reworded templates are **not** revisioned (they are
user-owned editable files), so they do not fast-forward into a project - the new wording guides how you
maintain those files going forward, and only newly created projects get the updated templates verbatim. The
read-first hook lives in the framework repo's own `.claude/settings.json` / `.cursor/hooks.json`, not in any
ai-pack.

## Migrate

1. **Stamp the version:** `uv run -m solaris.tools.version set --dir <project> 0.12.0`.
2. **Re-baseline** (no-op for content, keeps the ledger tidy): `uv run -m solaris.tools.revs baseline --dir <project>`.
3. **(Optional) adopt the inventory/how split** in this project's own files: move any build/run/deploy/restart
   procedures, model/runtime details, or gotchas out of `ai/memory/resources.md` into
   `ai/engineer.instructions.md` (referencing `resources.md` for concrete host/path/port values), leaving
   `resources.md` as a flat inventory of what exists. Lossless reword only.

## Summary

0.2.0 trims a project root to just `AGENTS.md` (both IDEs read it natively - no `CLAUDE.md`, no
`.cursor/rules` shim), keeps MCP only as the gitignored runtime configs, and adopts per-file **revisions**
for ai-setup sync. This migration removes the now-unused root files, ensures the runtime MCP exists, and
records the revisions baseline. (None of this touches `src/`.)

## Pre-flight checks

- [ ] `projects/<slug>/ai/manifest.json` exists and is valid JSON.
- [ ] The framework is at 0.2.0 (`uv run -m solaris.tools.version current`).

## Migrate

1. Ensure the runtime MCP exists: if `projects/<slug>/.mcp.json` is missing but `mcp.json.example` is
   present, copy `mcp.json.example` -> `.mcp.json` and `.cursor/mcp.json`
   (`uv run -m solaris.tools.mcp_sync --dir projects/<slug> --sync --from example`).
2. Remove the now-unused root files: `projects/<slug>/CLAUDE.md`, `projects/<slug>/mcp.json.example`,
   `projects/<slug>/.gitignore`, and `projects/<slug>/.cursor/rules/` (keep `.cursor/mcp.json`).
3. Add `"revisions": {}` to `projects/<slug>/ai/manifest.json` if absent.
4. Materialize rev-marked masters and record the baseline:
   `uv run -m solaris.tools.revs ff --dir projects/<slug>` (in-sync files are re-copied with their rev
   markers; missing ones are added), then `uv run -m solaris.tools.revs baseline --dir projects/<slug>`.
5. Stamp the version: `uv run -m solaris.tools.version set --dir projects/<slug> 0.2.0`.

## Validate

- [ ] `projects/<slug>/` has no `CLAUDE.md`, `mcp.json.example`, `.gitignore`, or `.cursor/rules/`.
- [ ] `projects/<slug>/.mcp.json` and `.cursor/mcp.json` exist and `mcp_sync --check` is clean.
- [ ] `ai/manifest.json` has a non-empty `revisions` map and `framework_version` is `0.2.0`.
- [ ] `uv run -m solaris.tools.revs classify --dir projects/<slug>` reports all `in-sync`.

## Revert

1. `uv run -m solaris.tools.version set --dir projects/<slug> 0.1.0`.
2. Restore the 0.1.0 root files from the 0.1.0 template (`CLAUDE.md`, `mcp.json.example`, `.gitignore`,
   `.cursor/rules/<slug>.mdc`) and remove the `revisions` map from `ai/manifest.json`. (Materialized file
   contents are unchanged apart from their rev markers, which 0.1.0 ignores.)

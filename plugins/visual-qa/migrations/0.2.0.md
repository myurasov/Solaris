## Summary

0.2.0 reworks the shared eyes client and the serving story. `eyes.py` gains a serving-instance registry
(`use` / `pick` and per-task instance resolution), native video ingestion (fps sampling, chunked
re-encode, video data URLs) and a `watch` tool for temporal assertions; `models.json` is expanded and
re-ranked. Serving moves from a single `serve.sh` to lifecycle scripts (`install.sh` / `start.sh` /
`status.sh` / `stop.sh` / `uninstall.sh`) that keep everything on the host under `~/.solaris/visual-qa/`
and support multiple resident instances. The materialized copies in `ai/visual-qa/` must be refreshed.

## Pre-flight checks

- [ ] `ai/manifest.json` lists `visual-qa` in `plugins[]`.
- [ ] `ai/visual-qa/` exists (the plugin was installed, not just referenced).

## Migrate

1. Copy `plugins/visual-qa/shared/eyes.py`, `shared/models.json`, `shared/visual-qa.skill.md`, and
   `shared/visual-qa.rule.md` over the copies in `ai/visual-qa/`.
2. Set the plugin entry's `version` to `0.2.0` in `ai/manifest.json` `plugins[]`.
3. Keep the project's configured `VISUAL_QA_ENDPOINT` / `VISUAL_QA_MODEL` values as-is (they were chosen
   at install time and recorded in `ai/memory/resources.md`).
